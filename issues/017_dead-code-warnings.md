# 未使用コード（dead_code）警告の対処法

## 概要

Rustの`cargo check`で発生する未使用コード警告の適切な対処法。

## 対処方針

### 削除ではなく警告抑制を選択するケース

1. **将来使用予定のコード**
   - 再開機能、状態確認機能など、実装予定の機能で使用するメソッド
   - 例: `next_page_token()`, `set_next_page_token()`, `is_running()`

2. **外部APIレスポンスのフィールド**
   - YouTube APIなど外部サービスのレスポンスに含まれるフィールド
   - 削除するとデシリアライズが失敗する可能性あり
   - 例: `VideoStatistics::comment_count`

3. **ヘルパーメソッド**
   - 将来のエラーハンドリング強化時などに使用予定
   - 例: `SaveCommentsResult::is_complete()`

### 実装パターン

```rust
/// メソッドの説明
/// NOTE: 現在未使用だが、[将来の用途]で使用予定（PR#XXX: [機能名]）
#[allow(dead_code)]
pub fn method_name(&self) -> ReturnType {
    // 実装
}
```

**ポイント**:
- NOTEコメントで将来の用途を明記
- 可能であればPR番号やタスク番号を参照
- 後から見ても意図が分かるようにする

## 注意事項

### テストコードからの参照

- `#[cfg(test)]`モジュール内からの参照は、`cargo check`の警告抑制にカウントされないことがある
- テストで使用しているメソッドでも警告が出る場合は`#[allow(dead_code)]`が必要

### 確認事項

- 警告抑制する前に、本当に将来使用する予定があるか確認
- 使用予定がない場合は削除を検討
- PR番号やタスク番号を参照する場合は、それが有効であることを確認
